# Домашнее задание к занятию 4 «Работа с roles»

В рамках данного домашнего задания была выполнена декомпозиция монолитного плейбука на отдельные независимые роли. Это позволяет повторно использовать код и изолировать конфигурацию каждого компонента системы.

## 📁 Структура проекта в папке 04

```text
Ansible/04/
├── group_vars/
│   └── clickhouse/
│       └── vars.yml          # Переменные для ClickHouse
├── inventory/
│   └── prod.yml              # Файл инвентаря с хостами
├── README.md                 # Этот отчет о проделанной работе
├── requirements.yml          # Манифест внешних зависимостей (ролей)
└── site.yml                  # Главный управляющий плейбук
```

## 🛠 Выполненные работы

1. **Создание и публикация роли Vector (`vector-role`)**:
   * Инициализирована структура роли с помощью `ansible-galaxy role init`.
   * Переменные разделены по приоритетам: версии и ссылки вынесены в `defaults/main.yml`, а системные пути установки зафиксированы в `vars/main.yml`.
   * Конфигурация агента переведена на динамический шаблон `vector.toml.j2` (автоматически определяет IP кликхауса из инвентаря).
   * Код опубликован в отдельном репозитории, составлено описание параметров в `README.md` и проставлен семантический тег `v1.0.0`.

2. **Создание и публикация роли LightHouse (`lighthouse-role`)**:
   * Настроена автоматическая установка веб-сервера Nginx и пакета `unzip`.
   * Реализована скачка master-ветки LightHouse, её распаковка в целевую директорию веб-сервера.
   * Конфигурация виртуального хоста Nginx вынесена в шаблон `lighthouse.conf.j2`.
   * Роль выложена в отдельный репозиторий, снабжена документацией и тегом `v1.0.0`.

3. **Связывание ролей в главном плейбуке (`site.yml`)**:
   * В корне папки `04` создан файл `requirements.yml`, описывающий зависимости от трех внешних репозиториев (ClickHouse от Сетевого, а также собственные Vector и LightHouse).
   * Главный файл `site.yml` полностью очищен от задач (`tasks`) и переписан на лаконичный вызов через блок `roles`.
   * Локальное скачивание и работа зависимостей проверены командой `ansible-galaxy install -r requirements.yml -p ./roles`.
