# Домашнее задание к занятию 3 «Использование Ansible»

Репозиторий содержит Ansible-плейбук для автоматического развертывания и связывания аналитического конвейера сбора логов (Log Pipeline) в составе: СУБД **ClickHouse**, логирующего агента **Vector** и веб-интерфейса **LightHouse** на трех независимых хостах в Yandex Cloud.

---

## Что делает Playbook

Плейбук `site.yml` осуществляет автоматическую конфигурацию трех целевых окружений:

1. **Install Clickhouse**:
   * Скачивает официальные пакеты СУБД ClickHouse (`clickhouse-server`, `clickhouse-client`, `clickhouse-common-static`) заданной версии.
   * Устанавливает пакеты на целевую ОС Ubuntu с автоматическим разрешением циклических зависимостей пакетов.
   * Создает базу данных `logs` и разворачивает плоскую таблицу `syslog` для записи входящих системных событий.
   * Конфигурирует сервер для приема внешних сетевых подключений.

2. **Install and configure Vector**:
   * Скачивает бинарный архив Vector Log Agent напрямую из официального репозитория GitHub.
   * Распаковывает, устанавливает бинарный файл в систему и настраивает Systemd-сервис для автоматического перезапуска.
   * Разворачивает конфигурационный файл `vector.toml` из Jinja2-шаблона.
   * Настраивает сбор логов из файлов `/var/log/syslog` и `/var/log/auth.log` с фильтрацией собственных логов (для исключения эффекта бесконечной петли записи) и отправляет их по HTTP-протоколу напрямую в ClickHouse.

3. **Install and configure LightHouse**:
   * Производит установку веб-сервера Nginx и утилиты распаковки архивов `unzip`.
   * Скачивает актуальный master-архив интерфейса LightHouse из GitHub.
   * Распаковывает статические файлы приложения напрямую в веб-директорию `/var/www/lighthouse`.
   * Разворачивает виртуальный хост Nginx из шаблона, активирует сайт и перезапускает веб-сервер.

## Параметры и переменные (Variables)

Основные параметры конфигурации вынесены в переменные внутри сценариев для обеспечения гибкости:

| Имя переменной | Значение по умолчанию | Описание |
| :--- | :--- | :--- |
| `clickhouse_version` | `"23.3.8.21"` | Версия скачиваемых и устанавливаемых DEB-пакетов ClickHouse |
| `vector_version` | `"0.40.0"` | Версия скачиваемого агента сбора логов Vector |
| `vector_install_dir` | `"/opt/vector"` | Путь к директории установки бинарных файлов Vector |
| `vector_config_dir` | `"/etc/vector"` | Путь для размещения конфигурационных файлов Vector |
| `lighthouse_dir` | `"/var/www/lighthouse"` | Целевой каталог для размещения статических файлов веб-интерфейса |

## Теги и ограничение выполнения (Tags & Limits)

Для выборочного запуска отдельных частей инфраструктуры или конкретных сценариев (Plays) используются ограничения по хостам, указанным в inventory-файле `prod.yml`:

* `--limit clickhouse` — запускает только сценарий установки и настройки СУБД ClickHouse.
* `--limit vector_servers` — запускает только сценарий установки и настройки агента Vector.
* `--limit lighthouse` — запускает только сценарий развертывания веб-интерфейса LightHouse и Nginx.

## Инструкция по запуску и проверке

### 1. Проверка синтаксиса и оформления (Linter)
Перед запуском рекомендуется выполнить статический анализ кода плейбука на соответствие правилам Ansible Best Practices:
```bash
ansible-lint site.yml
```

### 2. Тестовый прогон (Симуляция изменений)
Запуск плейбука с флагом `--check` для проверки доступности хостов и валидации логики тасок без изменения состояния целевых систем:
```bash
ansible-playbook -i inventory/prod.yml site.yml --check
```

### 3. Боевой запуск (Развертывание инфраструктуры)
Запуск плейбука в режиме отображения вносимых изменений в конфигурационные файлы:
```bash
ansible-playbook -i inventory/prod.yml site.yml --diff
```

### 4. Проверка идемпотентности
Повторный запуск боевой команды. Критерием успешности и полной идемпотентности разработанного плейбука является строгое равенство счетчика изменений нулю (`changed=0`):
```bash
ansible-playbook -i inventory/prod.yml site.yml --diff
```

## Проверка работоспособности конвейера

1. **Проверка получения данных в ClickHouse:**
   Выполните HTTP-запрос к СУБД ClickHouse для проверки счетчика записей логов, поступивших от Vector:
   ```bash
   curl -s -d "SELECT count() FROM logs.syslog" http://<IP_ВАШЕГО_CLICKHOUSE>:8123
   ```
   *Счетчик должен возвращать число больше нуля и непрерывно расти.*

2. **Проверка веб-интерфейса LightHouse:**
   Откройте в браузере IP-адрес хоста `lighthouse` (порт 80). В окне подключения укажите строку соединения с вашей базой данных: `http://<IP_ВАШЕГО_CLICKHOUSE>:8123`. Вы увидите интерактивную панель управления с вашей базой данных `logs` и поступающими системными событиями.
