## Задача 2

Выберите один из вариантов платформы в зависимости от задачи. Здесь нет однозначно верного ответа так как все зависит от конкретных условий: финансирование, компетенции специалистов, удобство использования, надежность, требования ИБ и законодательства, фазы луны.

Тип платформы:

- физические сервера;
- паравиртуализация;
- виртуализация уровня ОС;

Задачи:

- высоконагруженная база данных MySql, критичная к отказу;
- различные web-приложения;
- Windows-системы для использования бухгалтерским отделом;
- системы, выполняющие высокопроизводительные расчёты на GPU.

Объясните критерии выбора платформы в каждом случае.

Решение:

1. Высоконагруженная БД MySQL / 4. Системы с GPU:Выбор:
      Паравиртуализация (в составе отказоустойчивого кластера).
      Обоснование: С точки зрения обеспечения доступности (SLA) и непрерывности бизнеса (BCP), виртуализация является приоритетной.Кейс из практики: Использование кластерных решений (как в моем опыте с Fusion Compute) позволяет нивелировать аппаратные сбои (например, выход из строя CPU). В случае с «голым железом» (Bare Metal) время простоя при аппаратной аварии будет критическим.
      Масштабируемость: Виртуализация позволяет реализовать репликацию между территориально распределенными ЦОД, что практически невозможно или крайне сложно реализовать на чистых физических серверах без существенных затрат и простоев.
   
2.  Различные web-приложения
      Виртуализация уровня ОС (Контейнеризация / Docker).
      Обоснование:Плотность размещения (Density): Web-приложения (особенно микросервисы) часто потребляют мало ресурсов. Контейнеризация позволяет запустить десятки приложений на одном ядре ОС без оверхеда на запуск полноценных гостевых систем. Мы экономим RAM и CPU.
      Изоляция зависимостей: У одного приложения нужен Python 3.9, у другого 3.12. В контейнерах они живут в своих «песочницах» и не конфликтуют библиотеками, при этом используя одно общее ядро хоста.
      Скорость и CI/CD: Контейнер запускается за доли секунды. Это критично для процессов автотестирования и мгновенного развертывания новых версий (Blue-Green deployment).
      Переносимость (Portability): Мы избавляемся от проблемы «на моем компе работало, а на сервере нет». Образ контейнера идентичен в любой среде — от ноутбука разработчика до продакшн-кластера.
    
3. Windows-системы для бухгалтерии / ИТР
      Физические сервера (рабочие станции).
      Обоснование: С учетом текущей рыночной ситуации в РФ и высокой стоимости VDI-решений (лицензирование, специализированное серверное железо), использование локальных рабочих станций является экономически более оправданным. Особенно это критично для рабочих мест с использованием CAD-систем (инженеры-технологи), где виртуализация графических мощностей (GPU) требует избыточных затрат при сомнительном приросте производительности.

   
## Задача 3

Выберите подходящую систему управления виртуализацией для предложенного сценария. Опишите ваш выбор.

Сценарии:

1. 100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий.
2. Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.
3. Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.
4. Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.
